# turn-mcp-web 技能

你有 `turn.wait` 工具（别名：`turn_wait`、`turn`）。在任务的每个步骤都用它与人工沟通。

## 它做什么

调用 `turn.wait` 会暂停你的执行，把消息发送到人工的浏览器，然后阻塞直到他们回复。一次 API 会话变成无限轮的对话。**永远不要自行结束循环**——每次操作之后继续调用，直到人工说停。

## 调用格式

```
turn.wait(
  context   = "你做了什么 + 你即将做什么",   # 必填
  question  = "你需要人工回答的具体问题",     # 可选
  options   = ["选项A", "选项B", "选项C"],  # 可选 — 预设回复按钮
  agentName = "你的名称"                      # 可选
)
```

## 三个技巧

**1. 用 `options` 预设回复**  
当下一步有明确分支时，给人工按钮而不是让他们打字。他们点一下，你拿到文本继续执行。

```python
turn.wait(
  context="发现 3 个过期文件，共 240 KB。",
  question="怎么处理？",
  options=["全部删除", "先给我看看", "跳过"]
)
```

**2. 写给没在旁边看的人**  
每条消息都是冷读的。说清楚：你发现了什么、做了什么、接下来是什么。

**3. 遵守强化后缀**  
每次回复末尾都有提醒你再次调用 `turn.wait` 的后缀。遵守它——那个后缀就是会话保持存活的机制。

## 循环

```
做事 → turn.wait → 收回复 → 做事 → turn.wait → 收回复 → ...
```

由人工结束会话，你不要自行结束。
